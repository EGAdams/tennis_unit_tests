Edge Case Testing Strategy

Introduction

The following document outlines the edge case testing strategy for the C++ code provided. Edge case testing ensures that software behaves as expected even in extreme or unanticipated situations.

Test Scenarios

1. TestFirstScore:

  • Checks the initial scores for both players.
  • Increments the score for player 1 and verifies the updated score on the scoreboard.

2. TestPlayerPointsEqualOpponentPoints:

  • Tests behavior when both players have the same score.
  • Both players' scores are set to 3 and the updated scores on the scoreboard are verified.

3. TestPlayerWinsGame:

  • Simulates a scenario where player 1 wins a game.
  • Player 1's score is set to 5, and player 2's score is set to 3.
  • Verifies that the game count for player 1 is incremented.

4. TestMode1P1Score_3Points:

  • Checks the behavior when player 1 has a score of 3 and player 2 has a score of 2.
  • Verifies the scores on the scoreboard and ensures they match the expected format.

5. TestDeuceScenario:

  • Simulates the scenario when both players have a score leading to "deuce".
  • Both players' scores are set to 3 and the scoreboard is verified to reflect the deuce.

6. TestWinAfterAdvantage:

  • Verifies the scenario when a player wins after gaining an advantage following a deuce.
  • Player 1 is given the advantage and then scores another point to win the game.

Conclusion

This testing strategy aims to cover all potential edge cases in the game scoring system. By ensuring thorough testing, we can confidently present to investors that our software is robust, reliable, and handles all possible game scenarios appropriately.
